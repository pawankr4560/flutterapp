# Auth Feature

## Purpose
Owns authentication, session state, login, signup, and password reset flows.

## Responsibilities
- Present auth screens and widgets.
- Coordinate auth services and request models.
- Persist authenticated user session details.

## What Belongs Here
- Auth pages, providers, services, request models, and auth repositories.

## What Should NOT Be Placed Here
- Dashboard, loan, payment, or profile business rules.
